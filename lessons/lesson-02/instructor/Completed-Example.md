# Completed Example — ERP Classification and Value Chain

## Classification

| Work | Classification | Owner |
|---|---|---|
| Monthly financial close | Operation | CFO/Finance Process Owner |
| Temporary report correction | Operational improvement | Finance Systems Owner |
| ERP design, build, migration, UAT and cutover | Project | Sponsor and Project Manager |
| Hypercare then steady-state support | Transitional → Operation | PM then IT Operations Owner |

## Value Chain

| Stage | Statement | Evidence | Owner |
|---|---|---|---|
| Need | ข้อมูลไม่ตรงและปิดงบ 15 วัน | Current close calendar | CFO |
| Output | ERP 5 modules, migrated data, trained users | Acceptance records | PM |
| Outcome | ใช้ process/data กลาง | Adoption and data-quality metrics | Process Owners |
| Benefit | ปิดงบภายใน 5 วัน | Six-month benefits review | CFO |
| Value | ตัดสินใจเร็วขึ้นและรองรับการเติบโต | Executive KPI review | Executive Sponsor |

## Governance Recommendation

ตั้ง ERP เป็น Project และทำ short-term report improvement เป็นงานแยก โดยห้ามใช้ short-term fix แทน root-cause decision
