# Thinking Walkthrough — Project vs Operation Analysis

## Watch A: ดู PM คิด

1. เริ่มจาก Business Need ไม่เริ่มจากชื่อ solution
2. แยกงานตามธรรมชาติ: temporary/unique กับ ongoing/repetitive
3. ไม่ใช้ deadline เป็นเกณฑ์เดียว เพราะ operation ก็มี deadline
4. หา transition point ว่าเมื่อใด accountability ย้ายจาก project ไป operation
5. สร้าง Value Chain โดยบังคับให้แต่ละ stage มี measure และ Owner
6. ตรวจว่าผลลัพธ์เกิดจาก Output จริงหรือเป็นเพียง assumption

## ERP Reasoning

Monthly close เป็น Operation ส่วน ERP implementation เป็น Project การแก้รายงานชั่วคราวอาจเป็น operational improvement ที่เดินคู่กับ Project ได้

## Hotel Contrast

การสร้างและ launch platform เป็น Project/initiative ส่วนการรับ booking, support incident และ update inventory หลัง launch เป็น Operation/Product Operation
