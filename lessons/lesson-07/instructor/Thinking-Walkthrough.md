# Thinking Walkthrough — Requirement to WBS

```text
Business Objective
→ Stakeholder Need
→ Testable Requirement
→ Scope Boundary
→ Deliverable
→ WBS Level 1/2
→ Work Package
→ Acceptance Criteria
```

1. ยืนยัน Objective จาก Charter ไม่สร้าง objective ใหม่
2. ถาม decision/use case ก่อนเขียน report requirement
3. แยก product capability จาก project work
4. กำหนด exclusions เพื่อหยุด ambiguity
5. แตก WBS ตาม deliverable แล้วหยุดเมื่อ work package estimate/assign/control ได้
6. กิจกรรมอยู่ใน Lesson 08 ไม่ใช่ WBS ระดับล่างโดยอัตโนมัติ
