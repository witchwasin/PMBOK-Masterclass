# Lesson 08 Workshop — SIT Delay Recovery

## Scenario

ERP SIT เลื่อน 2 สัปดาห์ แต่ยังไม่รู้ว่า defect retest, migration rehearsal และ UAT ใช้ dependency ใดหรือมี float เท่าไร

## Task

1. Trace activities กลับ WBS
2. วาด dependency network แบบย่อ
3. ระบุ critical/near-critical path และข้อมูลที่ขาด
4. เสนอ crash, fast-track และ de-scope/resequence พร้อมผลต่อ cost/quality/risk
5. ระบุผู้อนุมัติ baseline change

## Constraint

ห้ามสรุปว่า go-live เลื่อนจากจำนวนวันที่ delay เพียงอย่างเดียว
