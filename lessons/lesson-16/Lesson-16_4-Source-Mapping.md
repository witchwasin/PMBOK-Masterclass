## 4. ERP Assumptions

- Predictive สำหรับ Contract, Infrastructure, Migration และ Cutover
- Iterative สำหรับ Process Design, Configuration และ User Feedback
- ออกแบบ Hybrid Governance

รายละเอียดเป็น Teaching Scenario ไม่ใช่ข้อเท็จจริงจากโครงการจริง

## 5. Hotel Booking Assumptions

- Agile สำหรับ UX, Feature และ Conversion
- Predictive Control สำหรับ Payment, Security, Compliance และ Launch
- เลือก Cadence ที่ต่างกันตาม Workstream

ระบบสมมติประกอบด้วย Mobile App, Customer Web App, Landing Page, Back Office, Booking Engine, Payment, Inventory, Notification และ Reporting

## 6. Content Boundaries

- ไม่อ้างว่าทุกองค์กรต้องใช้ Artifact เหมือนกัน
