# Thinking Walkthrough — Lifecycle and Process Group Map

## Watch A: ดู PM คิด

1. นำ Project boundary และ Value Chain จาก Lesson 02 มาใช้ ไม่สร้าง objective ใหม่
2. ออกแบบ phase จากลักษณะ delivery เช่น ERP อาจมี Blueprint, Build, Test, Cutover
3. วาง Process Groups ซ้ำภายในหลาย phase ตาม management need
4. จับคู่ Executing กับ monitoring signal เช่น migration execution กับ reconciliation result
5. กำหนด trigger ที่ต้อง replan ก่อนระบุวันที่ละเอียด
6. กำหนด Closing จาก acceptance, transition, open-item ownership และ benefits handoff

## Common Reasoning Error

การเขียน Initiating → Planning → Executing → Monitoring → Closing เป็นห้า phase ทำให้ทีมรอจนงานเสร็จจึงตรวจ performance ซึ่งสายเกินไป
