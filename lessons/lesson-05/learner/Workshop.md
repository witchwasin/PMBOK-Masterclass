# Lesson 05 Workshop — ERP Change Governance

## Scenario

SI ขอเพิ่ม Approval Workflow ราคา 2 ล้านบาทและเพิ่ม UAT 3 สัปดาห์ก่อน ERP go-live ข้อมูลที่ให้มายังไม่มี acceptance criteria, regulatory deadline และผลต่อ migration readiness

## Task

1. ตรวจว่า Charter/Governance ให้อำนาจใครตัดสินใจ
2. สร้าง Change Decision Record
3. ระบุ baselines และ stakeholders ที่ได้รับผลกระทบ
4. เสนอ include, defer และ alternative/de-scope พร้อม trade-off
5. ระบุ missing evidence และ decision deadline
6. ระบุ baseline updates หลังอนุมัติ

## Boundary

บทนี้สอน Change Governance:

```text
Request → Impact Analysis → Authority → Decision → Baseline Update
```

การบริหาร Risk Trigger, Issue Log และ Risk-to-Issue อยู่ใน Lesson 13

## Expected Output

Change Decision Record ที่ Steering Committee ใช้ตัดสินใจและตรวจสอบย้อนหลังได้
