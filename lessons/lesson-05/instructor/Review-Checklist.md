# Review Checklist — Integration Artifact Pack

## Charter

- [ ] Business Need/Objective ตรง Scenario Master
- [ ] Objective วัดได้และเชื่อม Outcome
- [ ] Scope เป็น high level ไม่ใช่ activity list
- [ ] Constraint และ assumption แยกกัน
- [ ] Sponsor, PM authority และ Benefit Owner ชัด

## Governance/Change

- [ ] Decision rights และ threshold ชัด
- [ ] Creator, Owner, Reviewer, Approver แยกกัน
- [ ] Change มี integrated impact และ ≥3 options
- [ ] Approval evidence ตรวจสอบย้อนหลังได้
- [ ] ระบุ baseline updates และ communication
- [ ] ไม่ใช้ Change Record แทน Issue Log
