# Traceability — Chapter 03

## Artifact Contract

| Field | Value |
|---|---|
| Artifact | Lifecycle and Process Group Map |
| Creator | Learner in Project Planner role |
| Artifact owner | Project Manager |
| Reviewer | PMO and Workstream Leads |
| Approval authority | Project Manager with Sponsor acknowledgement |
| Approval evidence | Reviewed lifecycle map and recorded governance acknowledgement |
| Minimum acceptance | Usable |

| Output | Required content |

## Acceptance Levels

| Level | Meaning |
|---|---|
| Incomplete | Treats Process Groups as rigid phases or lacks feedback/replanning |
| Usable | Separates phase from Process Group and identifies decisions, evidence and feedback loops |
| Professional | Connects lifecycle, governance gates, monitoring signals, closing evidence and transition |

## Handoff to Lesson 04

Lesson 04 uses this artifact to build `PM Coverage and Cross-impact Map`.

| Lesson 03 evidence | Lesson 04 use |
|---|---|
| Management activities | Identify which Knowledge Areas must be covered |
| Decision gates | Map authority and cross-area decision points |
| Feedback loops | Show cross-impact signals across scope, schedule, cost, quality, risk and stakeholders |
| Closing evidence | Preserve transition and benefit-readiness context |
| Lifecycle phases | Locate where Knowledge Area decisions recur |
