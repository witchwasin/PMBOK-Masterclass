# Traceability — Chapter 01

## Artifact Contract

| Field | Value |
|---|---|
| Artifact | PM Learning Baseline |
| Creator | Learner |
| Artifact owner | Learner |
| Reviewer | Instructor or AI Coach |
| Approval authority | Course Assessor |
| Approval evidence | Baseline rubric and assessor sign-off |
| Minimum acceptance | Usable |

## Inputs

| Input | Source |
|---|---|
| Learner experience and project assumptions | Learner-provided context |
| ERP Transformation Case v1.0 | `../../../scenarios/ERP-TRANSFORMATION-CASE.md` |
| Hotel Booking Platform Case v1.0 | `../../../scenarios/HOTEL-BOOKING-PLATFORM-CASE.md` |

## Outputs

| Output | Required content |
|---|---|
| PM Learning Baseline | Project context, current mental model, output-outcome assumptions, blind spots, evidence gaps and approval fields |

## Acceptance Levels

| Level | Meaning |
|---|---|
| Incomplete | Names only experience or goal without blind spot and evidence gap |
| Usable | States project context, blind spots, output/outcome assumptions and learning actions |
| Professional | Connects assumptions to scenario evidence, decision risk and how Lesson 02 will verify them |

## Handoff to Lesson 02

Lesson 02 uses the PM Learning Baseline to build `Project vs Operation Analysis and Value Chain`.

| Lesson 01 evidence | Lesson 02 use |
|---|---|
| Blind spots | Identify where project/operation boundary or value chain is unclear |
| Output assumptions | Test whether learner stops at deliverables |
| Outcome assumptions | Clarify adoption or operational change |
| Benefit assumptions | Connect measure, owner and review timing |
| Evidence gaps | Convert into questions before classifying project vs operation |
