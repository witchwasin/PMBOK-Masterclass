# Instructor Guide — Chapter 05

ลิงก์ไป Learner Chapter: [lesson-05-learner.md](./lesson-05-learner.md)

## Teaching Notes

สอน Integration เป็น decision system ไม่ใช่ document aggregation. ผู้เรียนต้องเห็นว่า PM รวม impact เพื่อให้ authority ตัดสินใจ ไม่ใช่ PM ตัดสินใจแทนทุกคน

## Learning Intent

ผู้เรียนต้องสร้าง artifact pack ที่ใช้เริ่มงานและ route change ได้จริง: Charter, Governance/Decision Rights Map และ Change Decision Record

## Thinking Walkthrough

1. Start with business need and cross-impact map
2. Separate issue, change request and decision
3. Identify baselines affected
4. Request missing evidence before recommendation
5. Offer at least 3 options
6. Identify approval authority and updates after decision

## Completed Example

ERP approval workflow change:

- Request: add multi-level approval before go-live
- Affected baselines: scope, schedule, cost, quality, resource, procurement, risk
- Missing evidence: regulatory urgency, SI estimate basis, data/UAT impact, acceptance criteria
- Options: approve now with baseline change, defer to phase 1.1, manual workaround with control
- Authority: Steering Committee / Change Authority
- Updates: baseline, decision log, stakeholder communication, test plan

## Model Answer

คำตอบ Professional ต้องไม่อนุมัติหรือปฏิเสธจากราคาและเวลาอย่างเดียว ต้องขอ regulatory urgency, acceptance criteria, SI estimate basis, data/migration impact, UAT capacity และ benefit consequence

Change มีผลต่อหลายด้านจึงต้องผ่าน authority ตาม threshold ไม่ใช่ PM ตัดสินใจลำพัง

## Review Checklist

- [ ] Charter มี need, objective, authority, boundary
- [ ] Governance map มี threshold และ evidence
- [ ] Change record มี impact, options และ recommendation
- [ ] แยก issue/change ได้
- [ ] มี baseline update และ communication หลัง decision
- [ ] ตัวเลข scenario ไม่ผิด

## Scoring Rubric

| Dimension | Incomplete | Usable | Professional |
|---|---|---|---|
| Charter | ขาด need/authority/boundary | ใช้อนุมัติเริ่มงานได้ | objective, value, authority traceable |
| Governance | ไม่รู้ใครตัดสินใจ | มี roles/threshold | cadence, escalation, evidence ครบ |
| Change analysis | ดู scope อย่างเดียว | ครอบคลุม baselines สำคัญ | เชื่อม impacts, benefits, options, risk |
| Decision | ไม่มี authority/evidence | มี decision record | baseline update และ communication ปิดวงจร |
| Boundary | ปน issue/change | แยกได้ | เชื่อม issue to request โดยไม่ทำซ้ำ process |

## Debrief Questions

- Decision นี้ PM ตัดสินใจเองได้หรือไม่
- Evidence ใดทำให้ recommendation เปลี่ยน
- ถ้า approve แล้ว baseline ใดต้อง update
- Stakeholder ใดต้องรู้ decision ภายใน 24 ชั่วโมง

## Remediation Guidance

ถ้าผู้เรียนเขียนแค่ “อนุมัติแล้ว” ให้บังคับเติม impact, authority, decision evidence และ baseline updates

## Artifact Approval Guidance

ผ่าน `Usable` เมื่อ artifact pack ใช้ authorize project และ route change ได้ ผ่าน `Professional` เมื่อ trace value, threshold, impact, approval และ update ครบ
