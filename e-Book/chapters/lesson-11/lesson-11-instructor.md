# Instructor Guide - Chapter 11

ลิงก์ไป Learner Chapter: [lesson-11-learner.md](./lesson-11-learner.md)

## Teaching Notes

สอนจาก bottleneck ไม่ใช่ staffing list. ERP UAT ต้องการ key-user capacity ในขณะที่ BAU ยังเดินอยู่

## Learning Intent

ผู้เรียนต้องสร้าง resource plan ที่ commit ได้จริง มี RACI ที่ Accountable ชัด และรู้ว่าใครอนุมัติ capacity

## Thinking Walkthrough

1. Start with quality/test workload
2. Identify role, skill and capacity needs
3. Ask who can commit departmental capacity
4. Create RACI with one accountable owner per decision
5. Identify exception or priority conflict
6. Update resource plan and prepare L12 communication audiences

## Completed Example

ERP UAT: Data Lead is Accountable for reconciliation; key users are Responsible at agreed capacity; Finance Functional Manager signs capacity; PM approves consolidated plan; Steering Committee handles priority conflict only when BAU/project priority changes

## Model Answer

Recommend phased UAT with named backfill option. The plan is usable only after each Functional Manager signs capacity and the RACI has one accountable role for each acceptance decision

## Review Checklist

- [ ] Role, person, assignment and capacity are distinct
- [ ] RACI has one accountable role per decision
- [ ] Functional managers commit capacity
- [ ] Exceptions are escalated only when appropriate
- [ ] Physical resources are considered
- [ ] Artifact can feed Lesson 12 audiences and escalation path

## Scoring Rubric

| Level | Description |
|---|---|
| Incomplete | Missing accountable roles, capacity or approval evidence |
| Usable | RACI and committed capacity support UAT execution |
| Professional | Capacity conflict, backfill, escalation and communication handoff are traceable |

## Artifact Approval Guidance

ผ่าน `Usable` เมื่อทุก accountable role และ capacity commitment ใช้ action ได้จริง พร้อม RACI sign-off
