# PMBOK Masterclass — PMBOK Edition Position

> **Pass 0 Deliverable** — กำหนดจุดยืนเรื่อง PMBOK Edition ของหลักสูตร

---

## 1. Primary Edition

หลักสูตรนี้ใช้ **Canonical Source** (`references/PMBOK-Overview.md`) เป็นแหล่งอ้างอิงหลัก

Canonical Source สรุปมาจาก Presentation Deck ที่ครอบคลุม:
- **Project Management Overview** (PM concepts พื้นฐาน)
- **Predictive Project Management** ซึ่งจัดโครงสร้างตาม **PMBOK 6th Edition** (5 Process Groups × 10 Knowledge Areas)
- **Agile Project Management** (Agile Manifesto, Scrum, Kanban)
- **Waterfall vs Agile vs Hybrid** (Selection and Tailoring)

ดังนั้น course structure ปัจจุบัน (16 lessons แบ่งตาม Knowledge Areas) มีรากมาจาก **PMBOK 6th Edition process architecture** เป็นหลัก

---

## 2. ทำไมยังใช้ PMBOK 6 Concepts

| เหตุผล | คำอธิบาย |
|---|---|
| **Canonical Source เป็นฐาน PMBOK 6** | Presentation Deck ที่ใช้อ้างอิงจัดโครงสร้างตาม 5 Process Groups × 10 Knowledge Areas |
| **ใช้กันแพร่หลายในภาคปฏิบัติ** | องค์กรไทยจำนวนมากยังคงใช้กรอบคิด Process Groups + Knowledge Areas ในการบริหารโครงการ |
| **เหมาะกับการสอน PM เชิงปฏิบัติ** | Process-based structure ทำให้ผู้เรียนเห็นว่า "ต้องทำอะไรบ้าง" ได้ชัดเจน |
| **PMP Exam ยังใช้ concepts เหล่านี้** | แม้ PMP จะเปลี่ยนเป็น PMBOK 7 framework แต่ยังคง test ความเข้าใจ Knowledge Areas |

---

## 3. PMBOK 7 ใน Course นี้

PMBOK 7th Edition เปลี่ยนจาก process-based ไปเป็น **principle-based** approach:
- 12 Project Management Principles
- 8 Performance Domains
- Value Delivery System
- Models, Methods, and Artifacts (แทนที่ prescribed processes)
- Tailoring
- Systems Thinking

### สิ่งที่ Course นี้จะทำกับ PMBOK 7:

| Action | รายละเอียด |
|---|---|
| **กล่าวถึง Principles** | ใน L01 และ L02 ระดับภาพรวม ว่า PMBOK 7 เปลี่ยนมุมมองจาก "process ที่ต้องทำ" เป็น "principles ที่ต้องยึด" |
| **อธิบาย Value Delivery** | ใน L02 เมื่อพูดถึง Value Chain |
| **อธิบาย Tailoring** | ใน L05 (Integration) และ L16 (Approach Selection) |
| **ไม่ restructure course ตาม Performance Domains** | เพราะ Canonical Source ไม่ได้จัดโครงสร้างแบบนั้น |

## 4. วิธีนำเสนอศัพท์และแนวคิด

ทุกบทเรียนต้องใช้ **Source Classification Labels** ดังนี้:

| Label | ใช้เมื่อ |
|---|---|
| `[PMBOK]` | แนวคิดที่มาจาก PMBOK โดยตรง (ระบุ edition ถ้ามีความแตกต่าง) |
| `[PMBOK 6]` | แนวคิดที่เฉพาะเจาะจงกับ PMBOK 6th Edition (เช่น 49 processes) |
| `[PMBOK 7]` | แนวคิดที่เฉพาะเจาะจงกับ PMBOK 7th Edition (เช่น 12 Principles) |
| `[Best Practice]` | แนวปฏิบัติที่ดีทั่วไป ไม่ใช่กฎ PMBOK |
| `[Enterprise Practice]` | Pattern ที่พบในองค์กรจริง |
| `[Teaching Scenario]` | ตัวอย่างที่สร้างขึ้นเพื่อการสอน |
| `[Professional Opinion]` | ความเห็นเชิงวิชาชีพ ไม่ใช่ข้อเท็จจริง |

## 5. สิ่งที่หลักสูตรนี้ไม่อ้าง (What We Do NOT Claim)

1. **ไม่อ้างว่า 5 Process Groups × 10 Knowledge Areas คือโครงสร้างของ PMBOK 7** — นี่คือ PMBOK 6 structure ที่ยังคงมีคุณค่าเชิงปฏิบัติ
2. **ไม่อ้างว่า PMBOK 6 ล้าสมัย** — PMBOK 7 ไม่ได้ "แทนที่" PMBOK 6 แต่เปลี่ยนมุมมองจาก prescriptive เป็น principle-based
3. **ไม่อ้างว่า course นี้ครบตาม PMBOK 7 standard** — course นี้ใช้ Canonical Source เป็นขอบเขต ไม่ได้ cover PMBOK 7 ทั้งเล่ม
4. **ไม่อ้างว่า course นี้เพียงพอสำหรับสอบ PMP** — course นี้สอนการคิดและตัดสินใจแบบ PM ไม่ใช่ exam prep

## 6. Exam-Oriented vs Practice-Oriented

| ด้าน | Exam-Oriented | Practice-Oriented |
|---|---|---|
| จุดประสงค์ | ตอบข้อสอบได้ | ทำงานจริงได้ |
| เนื้อหา | Cover ทุก topic ที่ออกสอบ | เน้น Decision Thinking + Scenario |
| Assessment | MC ตาม exam format | Decision Case + Trade-off + Artifact Review |
| **จุดยืนของ course นี้** | — | **✅ Practice-Oriented** |

---

## 7. Summary Position Statement

> **PMBOK Masterclass** ใช้ Canonical Source ที่จัดโครงสร้างตาม PMBOK 6th Edition (5 Process Groups × 10 Knowledge Areas) เป็นฐานหลัก พร้อม label แนวคิดจาก PMBOK 7 (Principles, Value Delivery, Tailoring) ไว้อย่างชัดเจน
>
> Course นี้มุ่งสอน **PM Thinking & Decision Making** ในเชิงปฏิบัติ ไม่ใช่ exam preparation
>
> ทุกส่วนเนื้อหาต้องระบุ Source Classification Label เพื่อให้ผู้เรียนแยกได้เสมอว่า "นี่คือ PMBOK, Best Practice, Enterprise Practice, Teaching Scenario หรือ Professional Opinion"
